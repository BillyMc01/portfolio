Lost it? Report! System

Lost it? Report it! Website

Following is a prototype website created for the Final Year Project of the BSc Computing course at Greenwich University. The prototype is named "Lost it? Report it!" and includes a form for reporting property that is lost on London's bus and tube networks. The objective of the Lost it? Report it! prototype was to improve on some of the UI/UX faults that were discovered in existing systems for reporting property lost on public transport. The website was designed using WordPress which connects to a google-sheet using a google API.

Home Page

The home page includes the most recent story that the user was reading, this makes it easy to continue where the reader left off. The prototype also makes heavy use of Axure's dynamic pannels to display the content. This design choice was chosen due to the ability to swipe to access different pannel states, therefore suiting a mobile interface.

Stories Page

The stories page is where the user will go to access new stories and is accessible via swiping upwards on the home page. This page includes a list of recommended stories and a search section accessible via tapping the search button in the top right of the page.

Quizzes Page

Like with the stories page, the quizzes page is where the user will access quizzes relating to the stories that they have read. This page also includes a search button where the user searches for more stories.

Profile Info Page

The profile information page is accessible via tapping the profile icon in the bottom right of the screen. This is where te user can customise their profile and see the history of the stories and quizzes they have completed.

Settings Page

The settings page is accessibile by tapping the gear icon in the bottom right of the screen above the paint icon. This is where the volume is adjusted and accessibility features are accessed.

Colour Select Page

The colour select page is accessibile by tapping the paint icon above the profile icon in the bottom right of the screen. This is where the user can change the colour accent of the app if desired.

Story Mode

This is what the page will look like when a story is being read with the content consisting of a picture with a short text describing the image. This page includes a button to turn on a screen reader, tp access the quiz and to return to the home page.

Quiz Mode

The quiz mode page is similar in design to the story mode page but includes content for the quiz. The same buttons are used here but there is now an option to go back to the story.
